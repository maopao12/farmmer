模式实现方式优点适用场景

🔄 核心设计：DataCollector接口 — 模拟/真实切换

接口抽象 + 多实现，通过 application.yml 切换
🏫 校内模式 (application-mock.yml) 
data:
  collector: mock
  mock:
    interval: 5000  # 5秒生成一次数据
MockDataCollector 定时随机生成传感器数据 → 直接写入数据库 + 通过WebSocket推向前端
🏭 基地模式 (application-prod.yml) 
data:
  collector: mqtt
  mqtt:
    broker: tcp://192.168.1.100:1883
    topics: sensor/+/data
MqttDataCollector 订阅MQTT Topic → 接收真实硬件数据 → 写入数据库 + WebSocket推送
🔑 关键点：切换只需修改 application.yml 中的 data.collector 值 (mock/mqtt)，业务代码零改动。数据库表、API接口、WebSocket推送逻辑完全一致。
